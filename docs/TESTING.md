# テスト方法 (TESTING.md)

## 1. テスト概要

### 1.1 テスト種別

| 種別 | 説明 | 実施タイミング |
|------|------|---------------|
| 手動テスト | UIと機能の動作確認 | 実装完了時 |
| 互換性テスト | 様々なWebサイトでの動作確認 | リリース前 |
| 回帰テスト | 変更後の既存機能確認 | コード変更時 |

### 1.2 テスト環境

- Chrome バージョン: 88以上（最新推奨）
- OS: Windows / macOS / Linux
- テスト用Webサイト: 後述のテストページを使用

---

## 2. 手動テストケース

### 2.1 インストールテスト

#### TC-001: 拡張機能の読み込み

| 項目 | 内容 |
|------|------|
| 前提条件 | Chrome デベロッパーモードが有効 |
| 手順 | 1. `chrome://extensions/` を開く<br>2. 「パッケージ化されていない拡張機能を読み込む」をクリック<br>3. プロジェクトフォルダを選択 |
| 期待結果 | エラーなく拡張機能がリストに表示される |
| 確認ポイント | アイコン、名前、バージョンが正しい |

#### TC-002: 拡張機能の更新

| 項目 | 内容 |
|------|------|
| 前提条件 | 拡張機能がインストール済み |
| 手順 | 1. コードを変更<br>2. 拡張機能の更新ボタンをクリック |
| 期待結果 | エラーなく更新される |

---

### 2.2 DevToolsパネルテスト

#### TC-003: DevToolsパネル表示

| 項目 | 内容 |
|------|------|
| 前提条件 | 拡張機能がインストール済み |
| 手順 | 1. 任意のWebページを開く<br>2. F12でDevToolsを開く<br>3. 「Element Inspector」タブをクリック |
| 期待結果 | パネルが表示される |
| 確認ポイント | - タイトル「Element Inspector」が表示<br>- 「検査開始」ボタンが表示 |

#### TC-004: DevToolsパネルでのボタン切り替え

| 項目 | 内容 |
|------|------|
| 前提条件 | DevToolsパネルが開いている |
| 手順 | 1. 「検査開始」ボタンをクリック<br>2. 再度ボタンをクリック |
| 期待結果 | - 初期: 「検査開始」<br>- クリック後: 「検査終了」（赤色）<br>- 再クリック後: 「検査開始」 |

---

### 2.3 Popup UIテスト

#### TC-005: Popup表示

| 項目 | 内容 |
|------|------|
| 前提条件 | 拡張機能がインストール済み |
| 手順 | 拡張機能アイコンをクリック |
| 期待結果 | Popupが表示される |
| 確認ポイント | - タイトル「Element Inspector」が表示<br>- ボタンが表示 |

---

### 2.4 Inspectモードテスト

#### TC-006: ホバーハイライト

| 項目 | 内容 |
|------|------|
| 前提条件 | Inspectモードが開始されている |
| 手順 | 1. ページ上の要素にマウスを移動<br>2. 別の要素に移動<br>3. 要素外に移動 |
| 期待結果 | - ホバー中の要素に青いアウトライン<br>- 移動で前の要素のハイライト解除<br>- 要素外でハイライトなし |

#### TC-007: 要素クリック選択

| 項目 | 内容 |
|------|------|
| 前提条件 | Inspectモードが開始されている（DevToolsパネルから） |
| 手順 | 1. ページ上の要素をクリック<br>2. DevToolsパネルを確認 |
| 期待結果 | - 選択した要素の情報がパネルに表示される<br>- **パネルは閉じない** |

#### TC-008: リンククリック防止

| 項目 | 内容 |
|------|------|
| 前提条件 | Inspectモードが開始されている |
| 手順 | リンク要素をクリック |
| 期待結果 | ページ遷移が発生しない |

#### TC-009: ESCキーでキャンセル

| 項目 | 内容 |
|------|------|
| 前提条件 | Inspectモードが開始されている |
| 手順 | ESCキーを押す |
| 期待結果 | - Inspectモードが終了<br>- ハイライトが解除<br>- DevToolsパネルのボタンが「検査開始」に戻る |

#### TC-010: モード終了後の動作

| 項目 | 内容 |
|------|------|
| 前提条件 | Inspectモードを開始→終了 |
| 手順 | 1. リンクをクリック<br>2. 要素にホバー |
| 期待結果 | - 通常のリンク動作<br>- ハイライトなし |

### 2.5 要素情報表示テスト

#### TC-011: 基本要素の情報表示

| 項目 | 内容 |
|------|------|
| テスト対象 | `<div id="main" class="container active">` |
| 期待結果 | - タグ名: div<br>- ID: main<br>- クラス: container, active<br>- 子要素数: (実際の数) |

#### TC-012: ID無し要素

| 項目 | 内容 |
|------|------|
| テスト対象 | `<p class="text">` |
| 期待結果 | - タグ名: p<br>- ID: なし<br>- クラス: text |

#### TC-013: クラス無し要素

| 項目 | 内容 |
|------|------|
| テスト対象 | `<span id="label">` |
| 期待結果 | - タグ名: span<br>- ID: label<br>- クラス: なし |

#### TC-014: 属性無し要素

| 項目 | 内容 |
|------|------|
| テスト対象 | `<p>` |
| 期待結果 | - タグ名: p<br>- ID: なし<br>- クラス: なし<br>- 子要素数: 0 |

---

### 2.6 履歴機能テスト

#### TC-015: 履歴への追加

| 項目 | 内容 |
|------|------|
| 前提条件 | DevToolsパネルが開いている |
| 手順 | 1. 検査開始<br>2. 3つの異なる要素を順番にクリック |
| 期待結果 | - 履歴セクションが表示される<br>- 3件の履歴が表示される<br>- 最新が一番上 |

#### TC-016: 履歴からの再表示

| 項目 | 内容 |
|------|------|
| 前提条件 | 履歴に複数の要素がある |
| 手順 | 履歴の項目をクリック |
| 期待結果 | その要素の情報が要素情報セクションに表示される |

## 3. 互換性テスト

### 3.1 テスト対象サイト

以下のサイトで動作確認を行う:

| サイト | 特徴 | 確認ポイント |
|--------|------|-------------|
| https://example.com | シンプルなHTML | 基本動作 |
| https://google.com | 動的コンテンツ | イベント競合 |
| https://github.com | SPA | DOM更新後の動作 |
| https://yahoo.co.jp | 複雑なレイアウト | 深いネスト |
| ローカルHTMLファイル | file:// プロトコル | 権限確認 |

### 3.2 チェックリスト

各サイトで以下を確認:

- [ ] 拡張機能が有効化される
- [ ] ホバーハイライトが表示される
- [ ] クリックで要素選択できる
- [ ] 要素情報が正しく表示される
- [ ] サイトの機能に影響がない
- [ ] コンソールにエラーがない

## 4. テスト用HTMLファイル

プロジェクトに `test/test.html` を作成して使用:

```html
<!DOCTYPE html>
<html lang="ja">
<head>
  <meta charset="UTF-8">
  <title>Element Inspector テストページ</title>
  <style>
    .container { padding: 20px; }
    .box { border: 1px solid #ccc; padding: 10px; margin: 10px 0; }
  </style>
</head>
<body>
  <h1>テストページ</h1>

  <!-- TC-011: 基本要素 -->
  <div id="main" class="container active">
    <p>ID・クラス両方ある要素</p>

    <!-- TC-012: ID無し -->
    <p class="text description">IDがない要素</p>

    <!-- TC-013: クラス無し -->
    <span id="label">クラスがない要素</span>

    <!-- TC-014: 属性無し -->
    <p>属性がない要素</p>
  </div>

  <!-- リンクテスト -->
  <a href="https://example.com" id="test-link">テストリンク</a>

  <!-- ネスト構造 -->
  <div class="box level-1">
    Level 1
    <div class="box level-2">
      Level 2
      <div class="box level-3">
        Level 3
        <span>深いネスト</span>
      </div>
    </div>
  </div>

  <!-- 複数クラス -->
  <div class="btn btn-primary btn-large active">複数クラス要素</div>

  <!-- フォーム要素 -->
  <form id="test-form">
    <input type="text" class="input" placeholder="テキスト入力">
    <button type="submit" class="btn">送信</button>
  </form>
</body>
</html>
```

- [ ] TC-001〜TC-016 すべてPASS

- [ ] 5サイト以上で動作確認

### コード品質
- [ ] コンソールエラーなし
- [ ] console.log の削除
- [ ] 不要なコメント削除

- [ ] README更新
- [ ] バージョン番号更新

### その他
- [ ] アイコンが正しく表示
- [ ] 説明文が適切
